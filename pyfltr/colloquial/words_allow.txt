# 口語表現check用の許容パターン。
# words.txtと違い、本ファイルはエージェントが読み込んでも
# 問題ない。各エントリーは技術文書に定着した複合動詞・複合名詞であり、
# 検出パターンの語幹を含むがそれ自体は口語ではない。
#
# 仕組み:
#   pyfltrの内蔵linter`colloquial-check`は本ファイルの正規表現にマッチした部分文字列を
#   同長の空白へ置換（オフセット保持のため）した上でwords.txtの検査を行う。これにより
#   定着した複合語の中の動詞語幹で誤検出が起きるのを回避する。
#
# 形式:
#   - 1行に1つの正規表現（re.searchで判定するためfullmatch不要）
#   - 「#」始まり行と空行は無視
#   - 各パターンはマスク範囲を必要最小限にすること
#
# 運用方針:
#   - 初期は最小限に留め、運用中に誤検出された定着用語のみ追加する
#   - 削除はいつでも可能（denylist側の警告がやや増えるだけ）
#   - 口語的響きが残る複合語は対象外。
#     代替表現がある語（書き出し・受け継ぐ・立て直す・出来る・書き直す・組み入れる など）は許容しない
#   - エントリー追加時はtests/colloquial_check_test.pyへ当該パターンの許容ケース
#     （denylist側にマッチしつつ本ファイルでマスクされる文脈）を併せて追加する
#   - 活用形は文字クラスで漏れなく列挙する（例: `送出[しすさせず]`のように主要活用形を網羅する）。
#     未然形・連用形などの漏れがあると誤検出が頻発するため
#   - 他動詞形を登録する際は対応する自動詞形も合わせて登録する
#     （例: `切り替え`を登録する場合は`切り替わ[るっらりれろ]`も登録）。
#     片方のみだと自動詞活用が漏れて誤検出が出る
