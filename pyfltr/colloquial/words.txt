# コーディングエージェントは本ファイルを読み込まないこと。
# 内容をエージェントのコンテキストへ取り込むと検出語の生成確率が上がり、
# 本チェッカーの目的（口語的表現の生成抑止）と逆方向に作用する。
# 本ファイルはdenylist内容をソースコード（`.py`ファイル）に静的に埋め込まない設計のため
# 辞書ファイルとして分離し、pyfltrパッケージへ同梱する。
# pyfltrの内蔵linter`colloquial-check`（起動経路: `python -m pyfltr.colloquial`）が
# CLI起動時に本ファイルを動的に読み込み、検査対象ファイルの内容と照合する。
# 検出時は`path:line:col: [match] ...`形式で検出語を含む1行を出力する。
#
# 形式:
#   - 1行に1つの正規表現（re.searchで判定するためfullmatch不要）
#   - 「#」始まり行と空行は無視
#   - 文字クラス`効[くきけかこい]`方式で代表的な活用形だけをカバーし、
#     「効果」「効率」のような熟語は文字クラス対象外として誤検出を回避する
#   - 任意で「pattern\treplacement」形式（タブ区切り）でreplacement列を併記できる。
#     検出時のCLI出力に書き換え候補として表示される。タブを含まない行は候補なし扱いとする
#
# 運用方針:
#   - 初期は広めに登録し、運用しながらユーザーが追加・削除する
#   - 誤検出が頻発する複合語はwords_allow.txt側へ登録する
#   - エントリー削除時はテストが正例として参照していないか確認する。
#     現行テストはファイル先頭パターンに依存しないが、サンプル抽出で許容パターンと
#     衝突する語を探索するため、両ファイルの整合に注意する
#   - エントリー追加時はtests/colloquial_check_test.pyへ当該パターンの検出ケースを
#     必ず併せて追加する。allowlist側に同時追加する場合は許容ケースも併設する
#     （辞書再編・文字クラス展開規則の変更で意図せず無効化される回帰を検出するため）
